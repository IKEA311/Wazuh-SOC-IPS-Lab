## 一、设计思路

本次实验的核心目标是模拟真实的中小型企业内网环境，包含：

- **安全监控区**: Wazuh Server 集中采集所有资产日志
- **Web 业务区**: Ubuntu 靶机运行 Docker DVWA 作为漏洞靶场
- **内网终端**: Windows 7 作为域内客户端，部署 Sysmon 采集内核级日志
- **攻击测试区**: Kali 攻击机模拟外部威胁

## 三、各组件角色与交互

### 3.1 Wazuh Server（SIEM 核心）
- **IP**: `192.168.152.128`
- **组件**: Wazuh Manager + Wazuh Indexer + Wazuh Dashboard
- **部署方式**: Docker Compose（单节点 All-in-One）
- **职责**: 接收所有 Agent 日志、规则匹配、告警生成、API 响应

### 3.2 Ubuntu 靶机（目标代理）
- **IP**: `192.168.152.129`
- **系统**: Ubuntu Server 24.04
- **组件**: Wazuh Agent + Docker
- **业务**: DVWA 运行在 Docker 容器中，映射宿主机 8080 端口
- **日志**: Apache/Nginx 访问日志、auth.log、Docker 容器日志

### 3.3 Kali 攻击机
- **IP**: `192.168.152.130`
- **工具**: SQLMap、Metasploit Framework、weevely、nmap、Burp Suite
- **攻击目标**: Ubuntu 靶机 :8080/DVWA（SQLi、WebShell）、Win7（MS17-010）

### 3.4 Windows 7 靶机
- **组件**: Wazuh Agent + Sysmon
- **Sysmon 配置文件**: `scripts/sysmon-config.xml`
- **采集事件**: 进程创建（EventID 1）、网络连接（EventID 3）、文件创建（EventID 11）、注册表修改（EventID 13）
- **漏洞**: MS17-010 永恒之蓝（未打补丁）

### 3.5 日志流转路径

```
Kali 攻击 (192.168.152.130)
    │
    ├──► Ubuntu 靶机 (192.168.152.129)
    │       └──► Wazuh Agent 采集日志
    │               └──► UDP 1514 ──► Wazuh Server (192.168.152.128)
    │                                   └──► 规则匹配 → 告警
    │
    └──► Win7 靶机
            └──► Sysmon (内核级日志)
                    └──► Wazuh Agent 采集事件通道
                            └──► UDP 1514 ──► Wazuh Server
                                                └──► 规则匹配 → 告警 → 自动封禁
```

## 四、网络配置参考

### 4.1 虚拟机网络设置
所有虚拟机使用 **VMware Workstation** 的 **VMnet8（NAT 模式）**，确保在同一网段:

| 虚拟机 | 网络模式 | IP 地址 | 备注 |
|--------|---------|---------|------|
| Wazuh-Server | NAT (VMnet8) | `192.168.152.128` | 静态 IP |
| Ubuntu-Target | NAT (VMnet8) | `192.168.152.129` | 静态 IP |
| Kali-Linux | NAT (VMnet8) | `192.168.152.130` | 静态 IP |
| Win7-Target | NAT (VMnet8) | DHCP 或静态 | 需确保与同网段互通 |

### 4.2 Wazuh Server 防火墙放行策略
```
Wazuh Server 需要放行:
  55000/tcp - Wazuh Agent 注册
  1514/udp - Wazuh Agent 事件上报
  1515/tcp - Agent 密钥下发
  443/tcp  - Wazuh Dashboard (Web UI)
  9200/tcp - Wazuh Indexer REST API

Agent → Manager 方向: Agent 主动连接 Server，单向出站
```

### 4.3 Ubuntu 靶机端口
```
8080/tcp - DVWA Web 应用（Docker 映射）
```

---

> **📌 实际部署提示**: 4 台虚拟机建议内存分配：Wazuh-Server (8G)、Ubuntu-Target (2G)、Kali (4G)、Win7 (4G)，总计约 18G 内存。
