# 06 - 问题排查与调试

> 🐛 本文档记录了项目过程中遇到的所有 Bug、排查思路和解决方案。

## 一、Wazuh Server Docker 部署问题

### 1.1 Bug: vm.max_map_count 不足导致 Indexer 启动失败

**现象**: `docker-compose up -d` 后 wazuh-indexer 容器立即退出。

**日志**:
```
wazuh-indexer | ERROR: max virtual memory areas vm.max_map_count [65530] 
is too low, increase to at least [262144]
```

**解决方案**:
```bash
sudo sysctl -w vm.max_map_count=262144
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
docker-compose down -v
docker-compose up -d
```

### 1.2 Bug: Wazuh Dashboard 访问 443 端口超时

**现象**: 浏览器访问 `https://192.168.152.128` 长时间加载后超时。

**排查过程**:
```bash
docker-compose ps                 # 所有容器 Up
docker-compose port wazuh-dashboard 443  # 检查端口映射
sudo ufw status                   # 防火墙未放行 443
```

**解决方案**:
```bash
sudo ufw allow 443/tcp
sudo ufw reload
```

### 1.3 Bug: Docker 容器日志占用磁盘过大

**现象**: 运行 3 天后根分区磁盘使用率达到 85%。

**解决方案**: 配置 Docker 日志轮转:
```bash
# /etc/docker/daemon.json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
sudo systemctl restart docker
docker-compose up -d
```

## 二、Agent 注册与连接问题

### 2.1 Bug: Linux Agent 注册失败 - "Invalid authorization key"

**现象**:
```bash
sudo /var/ossec/bin/agent-auth -m 192.168.152.128 -p 1515 -A Ubuntu-Target
# 输出: ERROR: (1205) Invalid authorization key
```

**原因**: 之前用同名 Agent 注册过但未删除。

**解决方案**:
```bash
# 在 Wazuh Server 上删除旧 agent
docker exec -it wazuh-manager /var/ossec/bin/agent_control -R 001
docker exec -it wazuh-manager /var/ossec/bin/manage_agents -d 001

# 重新注册
sudo /var/ossec/bin/agent-auth -m 192.168.152.128 -p 1515 -A Ubuntu-Target
```

### 2.2 Bug: Windows Agent 状态为 Disconnected

**现象**: Dashboard 显示 Windows Agent 状态为 `Disconnected`。

**排查过程**:
```powershell
Get-Service WazuhSvc              # Running
Get-Content "C:\Program Files\ossec-agent\ossec.log" -Tail 30
# "ERROR: Cannot connect to manager at 192.168.152.128:1514"
```

**原因**: Windows 防火墙阻止了出站 UDP 1514 端口。

**解决方案**:
```powershell
New-NetFirewallRule -DisplayName "Wazuh Agent Outbound" `
  -Direction Outbound -Protocol UDP -LocalPort 1514 -Action Allow
Restart-Service WazuhSvc
```

## 三、Sysmon 相关问题

### 3.1 Bug: Sysmon 日志未显示在 Wazuh

**现象**: Win7 上 Sysmon 运行正常，但 Wazuh Dashboard 搜不到 Sysmon 事件。

**排查过程**:
```powershell
Get-Content "C:\Program Files\ossec-agent\ossec.conf" | Select-String "Sysmon"
# 没有输出 - 缺少 Sysmon 事件通道配置
```

**解决方案**: 在 `ossec.conf` 中添加:
```xml
<localfile>
  <location>Microsoft-Windows-Sysmon/Operational</location>
  <log_format>eventchannel</log_format>
</localfile>
```
```powershell
Restart-Service WazuhSvc
```

### 3.2 Bug: Sysmon 配置过度采集导致 Win7 卡顿

**现象**: Windows 7 靶机 CPU 占用率持续 30%+。

**解决方案**: 优化 `sysmon-config.xml`，缩小采集范围:
- 进程创建：仅监控 Temp、Downloads、可疑命令行
- 网络连接：仅监控 445/3389/4444 等关键端口
- 文件创建：仅监控脚本和可执行文件

## 四、渗透测试相关问题

### 4.1 Bug: SQLMap 检测不到注入点

**现象**: SQLMap 扫描全部返回 `[FALSE]`。

**排查过程**:
```bash
# 发现 SQLMap 获取不到正确的 Cookie，导致页面重定向
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1" -v 3
```

**解决方案**: 用浏览器登录 DVWA 获取有效 Cookie:
```bash
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="security=low; PHPSESSID=abc123" --batch
```

### 4.2 Bug: weevely WebShell 连接超时

**现象**: 执行 weevely 连接命令后卡住超时。

**排查过程**:
```bash
curl -v http://192.168.152.129:8080/hackable/uploads/shell.php
# 返回 200 OK，但内容为空
```

**解决方案**:
```bash
# 通过 Docker exec 修复文件权限
sudo docker exec -it dvwa chmod 755 /var/www/html/hackable/uploads/shell.php
```

## 五、自动化响应脚本问题

### 5.1 Bug: Wazuh API 认证失败

**现象**: Python 脚本返回 401 Unauthorized。

**排查过程**:
```bash
# 手动测试 API
curl -u wazuh-api:P@ssw0rdPurple! -k https://192.168.152.128:55000
# {"error": 401, "message": "Invalid credentials"}

# 确认 API 用户存在
docker exec -it wazuh-docker_single-node-wazuh-manager_1 /var/ossec/bin/wazuh-apid -l
```

**解决方案**: 进入正确的容器重新创建用户:
```bash
docker exec -it wazuh-docker_single-node-wazuh-manager_1 bash
/var/ossec/bin/wazuh-apid -u wazuh-api -p P@ssw0rdPurple! -a
```

### 5.2 Bug: SSH 连接 Ubuntu 靶机失败

**现象**: `paramiko.ssh_exception.NoValidConnectionsError`

**原因**: Ubuntu 靶机 SSH 服务未运行或密钥认证配置错误。

**解决方案**:
```bash
# 在 Ubuntu 靶机上
sudo apt install openssh-server -y
sudo systemctl enable ssh --now

# 配置密钥认证
ssh-keygen -t rsa -b 4096 -f /root/.ssh/auto_block -N ""
ssh-copy-id -i /root/.ssh/auto_block.pub root@192.168.152.129
```

## 六、综合调试技巧

### 6.1 日志速查路径

| 组件 | 日志路径 | 排查方向 |
|------|---------|---------|
| Wazuh Server | `/var/ossec/logs/ossec.log` | Agent 连接、规则匹配 |
| Wazuh API | `/var/ossec/logs/api.log` | API 认证和请求错误 |
| Ubuntu Agent | `/var/ossec/logs/ossec.log` | 日志采集和上报 |
| Windows Agent | `C:\Program Files\ossec-agent\ossec.log` | Windows Agent 连接状态 |
| Sysmon | Event Viewer → Sysmon/Operational | 内核级事件采集 |
| DVWA 容器 | `docker logs dvwa` | Web 访问日志 |

### 6.2 网络连通性测试

```bash
# 从 Kali 测试各主机
curl -I http://192.168.152.129:8080 2>/dev/null | head -1  # DVWA
nc -zv 192.168.152.131 445 2>&1                            # Win7 SMB

# 从各靶机测试到 Wazuh Server
nc -zu 192.168.152.128 1514                                 # Agent 上报
curl -k https://192.168.152.128:55000                        # API
```

### 6.3 规则调试速查
```bash
# 实时查看告警
docker exec -it wazuh-manager tail -f /var/ossec/logs/alerts/alerts.json

# 模拟攻击事件
echo "攻击日志内容" | logger

# 查看 Agent 状态
sudo /var/ossec/bin/agent_control -l
```
