## 一、Wazuh Server 部署（Docker 方式）

### 1.1 准备工作
```bash
# 宿主机: Ubuntu Server (Wazuh-Server, 192.168.232.132)
# 确保 Docker 和 Docker Compose 已安装
sudo apt update && sudo apt upgrade -y
sudo apt install docker.io docker-compose -y
sudo systemctl enable docker --now
```

### 1.2 下载 Wazuh Docker 部署包
```bash
# 克隆 Wazuh Docker 仓库
git clone https://github.com/wazuh/wazuh-docker.git -b v4.7.3
cd wazuh-docker/single-node
```

### 1.3 生成证书并启动
```bash
# 生成 SSL 证书
docker-compose -f generate-indexer-certs.yml run --rm generator

# 启动所有服务
docker-compose up -d

# 检查服务状态
docker-compose ps
```

> 📸 **截图**: `screenshots/wazuh-dashboard.png`
> 访问 `https://192.168.232.132:443`，使用默认凭据 `admin:admin` 登录 Wazuh Dashboard 后的主界面

### 1.4 🔧 踩坑记录：服务启动失败

**问题现象**: 执行 `docker-compose up -d` 后部分容器状态为 `Exit 1`。

**排查过程**:
```bash
docker-compose logs wazuh-indexer  # 查看索引器日志
# 发现: "max virtual memory areas vm.max_map_count [65530] is too low"
```

**解决方案**:
```bash
# 在宿主机上永久修改 vm.max_map_count
sudo sysctl -w vm.max_map_count=262144
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf

# 重建容器
docker-compose down -v
docker-compose up -d
```

## 二、Ubuntu 靶机环境搭建（192.168.232.135）

### 2.1 安装 Wazuh Agent
```bash
# 在 Ubuntu-Target 上安装 Wazuh Agent
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo apt-key add -
echo "deb https://packages.wazuh.com/4.x/apt/ stable main" | sudo tee /etc/apt/sources.list.d/wazuh.list
sudo apt update && sudo apt install wazuh-agent -y

# 配置 Manager 地址为 Wazuh Server IP
sudo sed -i 's/MANAGER_IP/192.168.232.132/g' /var/ossec/etc/ossec.conf

# 注册 Agent
sudo /var/ossec/bin/agent-auth -m 192.168.232.132 -p 1515 -A Ubuntu-Target

# 启动 Agent
sudo systemctl enable wazuh-agent --now
```

### 2.2 验证 Agent 连接
```bash
sudo /var/ossec/bin/agent_control -l
# 输出示例:
#   ID:001, Name:Ubuntu-Target, IP:192.168.232.135, Active
```

### 2.3 部署 DVWA（Docker 方式）
```bash
# 安装 Docker（如未安装）
sudo apt install docker.io -y
sudo systemctl enable docker --now

# 拉取并运行 DVWA Docker 容器
sudo docker run -d --name dvwa \
  -p 8080:80 \
  -e DVWA_WEB_PHP_OPLOG=false \
  vulnerables/web-dvwa

# 验证 DVWA 是否正常运行
sudo docker ps | grep dvwa
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080
# 返回 200 即正常
```

> 📸 **截图**: 浏览器访问 `http://192.168.232.135:8080` 显示 DVWA 登录页面的界面

### 2.4 配置 Wazuh Agent 采集 Apache/Docker 日志
确认 `/var/ossec/etc/ossec.conf` 中包含（Wazuh Agent 默认会采集系统日志，Docker 日志需额外配置）:

```bash
# 查看当前日志采集配置
sudo cat /var/ossec/etc/ossec.conf | grep -A 5 "<localfile>"
```

如需要采集 Docker 容器日志，添加:
```xml
<localfile>
  <location>/var/lib/docker/containers/*/*-json.log</location>
  <log_format>json</log_format>
</localfile>
```

重启 Agent:
```bash
sudo systemctl restart wazuh-agent
```

## 三、Windows 7 靶机环境部署

### 3.1 安装 Wazuh Agent（Windows 7）

1. 从 Wazuh 官网下载 Windows Agent 安装包（`.msi`）
2. 管理员身份运行安装：
   ```
   msiexec /i wazuh-agent-4.7.3-1.msi WAZUH_MANAGER=192.168.232.132
   ```
3. 如果自动注册失败，手动注册：
   ```powershell
   # 管理员 PowerShell
   & "C:\Program Files\ossec-agent\agent-auth.exe" -m 192.168.232.132 -p 1515 -A Win7-Target
   ```
4. 启动服务：
   ```powershell
   Set-Service -Name WazuhSvc -StartupType Automatic
   Start-Service -Name WazuhSvc
   ```

### 3.2 🔧 踩坑记录：Windows Agent 手动注册密钥

**问题现象**: Agent 安装时自动注册失败，Dashboard 显示 Agent 为 `Disconnected`。

**排查过程**:
```powershell
# 查看 Agent 日志
Get-Content "C:\Program Files\ossec-agent\ossec.log" -Tail 20
# 发现: "ERROR: (1205) Invalid authorization key"
```

**解决方案**: 在 Wazuh Server 上手动生成密钥并下发
```bash
# 在 Wazuh Server 上
docker exec -it wazuh-docker_single-node-wazuh-manager_1 /var/ossec/bin/manage_agents
```

选择选项:
```
1) Add agent
   Name: Win7-Target
   IP: 192.168.232.0/24
2) Extract key for agent
复制生成的密钥
```

将密钥写入 Windows Agent:
```powershell
echo "MDAxIFdpbjctVGFyZ2V0IDE5Mi4xNjguMTUyLjAvMjQgYWIzYzQ1ZDY3ODkwMTIzNDU2Nzg5MDEyMzQ1Njc4OTA=" | Out-File -FilePath "C:\Program Files\ossec-agent\client.keys" -Encoding ascii
Restart-Service -Name WazuhSvc
```

> 📸 **截图**: Wazuh Dashboard 中 Agent 注册成功后显示 Active 的界面

### 3.3 安装 Sysmon（Windows 7）

```powershell
# 下载 Sysmon 并安装（使用项目提供的配置文件）
# 将 sysmon-config.xml 复制到 Windows 7 上
.\Sysmon64.exe -accepteula -i .\sysmon-config.xml

# 验证 Sysmon 运行状态
Get-Service -Name Sysmon64
```

> 📸 **截图**: 在 Windows 7 上运行 `Get-Service Sysmon64` 显示 Running 状态的 PowerShell 窗口

### 3.4 配置 Windows Agent 转发 Sysmon 日志

编辑 `C:\Program Files\ossec-agent\ossec.conf`，添加以下内容:

```xml
<localfile>
  <location>System</location>
  <log_format>eventchannel</log_format>
</localfile>

<localfile>
  <location>Microsoft-Windows-Sysmon/Operational</location>
  <log_format>eventchannel</log_format>
</localfile>
```

重启服务:
```powershell
Restart-Service -Name WazuhSvc
```

## 四、验证整体连通性

### 4.1 连通性测试
```bash
# 从 Kali 测试到各主机的连通性
ping -c 3 192.168.232.132  # Wazuh Server
ping -c 3 192.168.232.135  # Ubuntu Target
```

### 4.2 Wazuh 端到端验证
在 Ubuntu 靶机上产生测试事件:
```bash
sudo logger -t "TEST_EVENT" "Purple-Team-Lab connectivity test"
```

在 Windows 7 上产生测试事件:
```powershell
Write-EventLog -LogName Application -Source "Test" -EntryType Information -EventId 9999 -Message "Purple-Team-Lab connectivity test"
```

然后在 Wazuh Dashboard → Security Events 中搜索 `Purple-Team-Lab` 确认日志到达。
